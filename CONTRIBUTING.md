# Contributing to ShopEase DevOps ## Branch Strategy - `main` ‚Äî production only. Never push directly. - `develop` ‚Äî integration branch. All features merge here. - `feature/SE-XX-description` ‚Äî your work branch. - `hotfix/SE-XX-description` ‚Äî urgent production fixes. ## Workflow 1. Pick Jira ticket ‚Üí move to "In Progress" 2. Branch off develop: `git checkout -b feature/SE-XX-name` 3. Make changes + commit with message: `type(SE-XX): description` 4. Push + raise PR to develop 5. PR must pass CI + 1 review 6. Merge ‚Üí move Jira ticket to Done ## Commit Types - `feat` ‚Äî new feature - `fix` ‚Äî bug fix - `chore` ‚Äî maintenance - `docs` ‚Äî documentation - `refactor` ‚Äî code restructure - `test` ‚Äî adding test
